00 · Registry Preview

A spellbook for agent-native capabilities.

SkillRune treats a skill as a package worth inspecting: instructions, files, compatibility, safety notes, and one clear install command.

Not a generic AI directory. Built for developers, agent builders, and teams sharing reusable workflows.

01 · What Is A Skill?

A reusable capability, packaged for an agent.

  1. InstructionsThe behavior contract.

    A skill tells an agent how to perform a specific task reliably: what to inspect, which tools to prefer, what to avoid, and how to verify the result.

  2. FilesThe working kit.

    A skill can include scripts, references, examples, templates, and checklists, so the capability is not trapped inside a single prompt.

  3. CompatibilityThe runtime edge.

    Each listing names the agents, models, tools, permissions, and local dependencies the skill expects before a developer installs it.

  4. Safety notesThe sharp edges.

    A serious registry makes risks visible: network access, file writes, secrets handling, browser sessions, destructive commands, and review status.

02 · Why SkillRune?

Because useful agent behavior should be inspectable.

Teams already pass around prompts, shell snippets, local scripts, and half-documented workflows. SkillRune turns that pile into a catalog with names, versions, authors, and install paths.

  1. 1 Developer-first listingsEvery skill page leads with what it does, what it touches, and how to install or audit it.
  2. 2 Curated before crowdedThe MVP favors a small, high-signal registry over a marketplace full of duplicate wrappers.
  3. 3 Agent-native metadataCompatibility, permissions, dependencies, and supported runtimes are first-class fields.
  4. 4 Install without ceremonyThe command pattern stays simple enough to copy, script, and eventually turn into a real CLI.

06 · Submit

A future intake path for reusable work.

A submitted skill should include its name, short description, category, tags, supported agents, repository URL, documentation URL, license, author, contact, and safety notes.

  1. PackageName it clearly.

    A good skill name should tell developers what capability they are equipping, not just which tool it uses.

  2. DocumentShow the contract.

    The listing should preview the README, file tree, install command, compatibility, and required permissions before download.

  3. ReviewEarn the catalog slot.

    For the MVP, submissions can be collected as structured intent and reviewed manually before appearing in the registry.

07 · FAQ

Questions before the registry opens.

How is SkillRune different from a prompt library?
A prompt library stores text. SkillRune is designed around installable skill packages with files, instructions, compatibility, safety notes, and versioned metadata.
Is this a marketplace?
Not in the MVP. The first version is a curated registry and demo catalog, without payments, accounts, or paid listings.
Does the install command work today?
No. skillrune install <slug> is the intended command pattern for the product experience, not a shipped CLI in this static page.
Which agents are supported?
The catalog model supports coding agents, research agents, browser agents, automation agents, and local CLI agents. Each skill should name its own support list.
Can teams submit private skills?
Private registries are out of scope for this landing page. The submit flow described here is for a future public review queue.
What makes a skill safe to install?
SkillRune should make permissions, file writes, network access, secrets handling, and destructive operations visible before the user downloads anything.

The full `/skills` and `/submit` routes can ship when the static landing page grows into the registry app.
